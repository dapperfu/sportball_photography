    • FR-001
    Unified Extraction Command FR-001

    The system shall provide a single 'sb extract' command that replaces both 'sb face extract' and 'sb object extract' commands.
    FR-001-1
    Primary Extraction Command FR-001.1

    The 'sb extract' command shall be the primary extraction command for all face and object extraction operations.
    FR-001-2
    Command Arguments FR-001.2

    The 'sb extract' command shall accept both input path and output directory as required arguments.
    FR-001-3
    Extraction Type Support FR-001.3

    The 'sb extract' command shall support extraction of faces, objects, or both types simultaneously.
    FR-001-4
    Feature Parity FR-001.4

    The 'sb extract' command shall maintain all existing functionality from individual face and object extraction commands.
    FR-001-5
    Help Documentation FR-001.5

    The 'sb extract' command shall provide clear help text and usage examples.
    FR-002
    Extraction Type Flags FR-002

    The system shall use flags to indicate what should be extracted from images.
    FR-002-1
    Face Extraction Flag FR-002.1

    The '--faces' flag shall extract only faces from images.
    FR-002-2
    Object Extraction Flag FR-002.2

    The '--objects' flag shall extract only objects from images.
    FR-002-3
    Combined Extraction Flag FR-002.3

    The '--both' flag shall extract both faces and objects from images.
    FR-002-4
    Default Behavior FR-002.4

    The default behavior shall be '--both' when no extraction flag is specified.
    FR-002-5
    Flag Mutual Exclusivity FR-002.5

    The extraction flags shall be mutually exclusive (only one can be specified at a time).
    FR-002-6
    Invalid Flag Error Handling FR-002.6

    The system shall show clear error messages for invalid flag combinations.
    FR-003
    Parameter Consolidation FR-003

    The system shall consolidate parameters from both extraction commands into the unified command.
    FR-003-1
    Required Output Directory FR-003.1

    The unified command shall accept 'output_dir' as a required argument.
    FR-003-2
    Face Padding Parameter FR-003.2

    The unified command shall support '--face-padding' for face extraction (default: 20px).
    FR-003-3
    Object Padding Parameter FR-003.3

    The unified command shall support '--object-padding' for object extraction (default: 10px).
    FR-003-4
    Object Type Filtering FR-003.4

    The unified command shall support '--object-types' for object filtering.
    FR-003-5
    Object Size Filtering FR-003.5

    The unified command shall support '--min-size' and '--max-size' for object size filtering.
    FR-003-6
    Common Parameters FR-003.6

    The unified command shall support '--workers', '--no-recursive', '--verbose', and '--show-empty-results'.
    FR-003-7
    Parameter Clarity FR-003.7

    Parameter names shall be clear and unambiguous.
    FR-004
    Output Directory Structure FR-004

    The system shall define a clear output directory structure for unified extraction.
    FR-004-1
    Face Output Directory FR-004.1

    When '--faces' is specified, faces shall be saved to '<output_dir>/faces/'.
    FR-004-2
    Object Output Directory FR-004.2

    When '--objects' is specified, objects shall be saved to '<output_dir>/objects/'.
    FR-004-3
    Combined Output Directories FR-004.3

    When '--both' is specified, faces shall be saved to '<output_dir>/faces/' and objects to '<output_dir>/objects/'.
    FR-004-4
    Automatic Directory Creation FR-004.4

    The directory structure shall be created automatically.
    FR-004-5
    File Conflict Resolution FR-004.5

    Existing files shall not be overwritten without explicit confirmation.
    FR-004-6
    Directory Structure Documentation FR-004.6

    The directory structure shall be documented in help text.
    FR-005
    Backward Compatibility FR-005

    The system shall maintain backward compatibility with existing extraction commands.
    FR-005-1
    Face Command Compatibility FR-005.1

    The 'sb face extract' command shall continue to work unchanged.
    FR-005-2
    Object Command Compatibility FR-005.2

    The 'sb object extract' command shall continue to work unchanged.
    FR-005-3
    Script Compatibility FR-005.3

    Existing scripts and workflows shall not break.
    FR-005-4
    Deprecation Notices FR-005.4

    Individual commands shall be marked as deprecated in help text.
    FR-005-5
    Migration Guide FR-005.5

    A migration guide shall be provided for users transitioning to the unified command.

    Error Handling and Validation FR-006

    The system shall provide clear error handling and parameter validation.

    Output Directory Validation FR-006.3

    Parameter Error Messages FR-006.4
